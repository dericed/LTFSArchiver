Introduction

ltfsa_gui IS-PART-OF LTFSArchiver package and provides a graphical user interface for the ltfsarchiver component of the package, which is the back-end server part.

Two main functionalities are provided: one for human operator of possible manually operated drives, with cassettes on shelves; one for an interactive use of the LTFSArchiver server APIs.

An alternative for both cases is the command line interface provided by ltfsa_cli, while for the latter it is possible to integrate the APIs into a wider, and possibly automated, archive management framework.

User guide

Manual Operator Interface

Using this interface the human operator can monitor the requested manual load operation for tapes not in the library. It is necessary to have at least one external drive. The request page is reloaded automatically.

In case of request the operator has to look for the LTO tape and insert for loading into the given LTO drive. Then she has to wait for the load to be complete (typically the green led on without flashing) and eventually confirm the successfull load on the interface.

Two failure situation can be handled: "Tape Not Found" and "Device Error".

APIs Interface

A human user can interactively use the LTFSArchiver server APIs through this interface. All the APIs are implemented. However in some cases they are offered to the user with criteria depending on the context; for example operations on tapes are possible from the results of a query on tapes and similarly with tasks and devices.

With the ltfsa_gui it is not possible to directly get XML or JSON output, as it is intended for HTML output. If you need XML or JSON output you will have to use the ltfsa_cli.

Administration guide

How to configure, install, and uninstall

You can install ltfsa_gui on the same server running ltfsarchiver or on another server, as you wish. Remember the all the file system paths given in input to ltfsarchiver are considered local to the server on which ltfsarchiver is running.

The package is implemented for Linux systems, it requires apache/httpd services, and the configuration process works AS IS on Ubuntu. It is possible to install ltfsa_gui on other Linux distributions as well, but you will need to modify the configuration script for the part regading the DEPENDENCY CHECK.

The detailed dependencies can be found in the configuration script.

The configuration script must be run before the installation and it will create a makefile for install and uninstall operations.

Instructions:

  • Unpack or copy the software folder to the desired working path (e.g. /opt or /usr/local)

  • cd ltfsa_gui

  • ./configure --help (you will get information about the configuration options)

  • ./configure [OPTIONS]

    You will be asked for giving the password for the operator

  • sudo make install

    'sudo make uninstall' will make the service unactive but will not remove the sofware

Changing configuration

Part of the configuration is about the installation and you will not need to modify it when running the service.

The two properties that might be changed when using the service are:

  • the URL of the ltfsarchiver service.

  • the options (for curl command) for connecting to the ltfsarchiver service.

Maintenance and troubleshooting

Operations are logged into logs folder and logs are never removed. Ask to your system administrator how to set up an automated purge of older logs, if desired; otherwise yyou will have to remove the logs manually

In case of problems, identify the reason by checking the following:

  • that the ltfsarchiver server is reachable and the service is up and running

  • look at the latest log file in the logs folder

  • look at the possible related error from the apache/httpd service (e.g. 'tail -f /var/log/apache2/error.log' on Ubuntu)
